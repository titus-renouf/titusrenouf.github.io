Working with an API with good practices

The goal of the project is to develop a tool ready to be used in production. We are working with the imdb API which is composed of many movie reviews. Our task is to develop the back-end app with three success criteria :
- The first page should be a list of the movie poster and name.
- On the first page we want to display a Navigation Box to search for movies by title. Then, the app is displaying the results.
- When selecting a movie its review is displayed.


The good practice is to create a python package for each request function, and then calling them in our program. Indeed we have created 5 requests functions and 1 file containing our API token. We are using it like that for the local development but we obviously should hide it as .env variable for the deployment. We are testing our code with a Notebook, so the following presentation is about my notebook results.

The get_top_movies function generates the names and posters of the current top movies. This corresponds to our main page.

The SearchMovies function is displaying a list of films corresponding to a research, here I am looking for films related to "Inception".
For this new page I chose to display the name, poster, description and rate of the movie.
To get the movie rate I have imported the get_movie_rate function from the MovieRate package.

The get_movie_review function displays a new page corresponding to the movie review where I chose to show the title, poster, release date, plot, runtime, directors, actors, trailer video and rate. Like for the previous function I used the get_movie_rate from the MovieRate package, but here I also used the get_movie_trailer function from the MovieTrailer package.
